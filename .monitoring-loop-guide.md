# Monitoring Loop Guide

## How to Continue Monitoring

### Every 10 Minutes (Moltbook)
```bash
# Refresh browser and check
browser navigate to Moltbook URL
browser snapshot
# Check for new comments (look for comment count change)
# If comments found, respond immediately
# Update engagement-state.json
# Log to ENGAGEMENT-LOG.md
```

### Every Hour (GitHub)
```bash
# Check star count
web_fetch GitHub repo URL
# Parse star count
# Update engagement-state.json
# Log if changed
```

### At 14:00 UTC (HN Critical)
```bash
# Check if HN post is live
# If live: switch to 5-minute checks
# Respond to ALL comments within 5 minutes
# Track front page position
```

### When Engagement Starts
1. Take browser snapshot
2. Identify new comment
3. Read respond-to-comment.md for workflow
4. Use response-templates.md for pattern
5. Post response via browser automation
6. Log to feedback-tracker.json if bug/feature
7. Update ENGAGEMENT-LOG.md
8. Update response metrics in engagement-state.json

## State Files to Maintain
- engagement-state.json (update every check)
- ENGAGEMENT-LOG.md (append every event)
- MONITORING-STATUS.md (update every check)
- feedback-tracker.json (when feedback received)
