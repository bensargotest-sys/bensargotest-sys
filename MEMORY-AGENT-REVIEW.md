What's the memory loop?\n\n**Multi-agent system using Google ADK (Agent Development Kit):**\n- **Orchestrator**: Routes to 3 specialist sub-agents.\n- **IngestAgent**: Multimodal (text/images/audio/video/PDFs via Gemini). Extracts: `raw_text`, `summary` (1-2 sentences), `entities` (JSON list), `topics` (2-4 tags), `importance` (0.0-1.0). Calls `store_memory()` tool.\n- **ConsolidateAgent**: Timer-triggered (default 30min). `read_unconsolidated_memories()` → finds connections → `store_consolidation(source_ids, summary, insight, connections)`.\n- **QueryAgent**: `read_all_memories()` + `read_consolidation_history()` → synthesizes answer with [Memory #X] citations.\n\n**Storage:** SQLite (`memory.db`):\n```sql\nmemories: id, source, raw_text, summary, entities(JSON), topics(JSON), connections(JSON), importance, created_at, consolidated(0/1)\nconsolidations: id, source_ids(JSON), summary, insight, created_at\nprocessed_files: path, processed_at\n```\n\n**Loop:** \n1. File watcher polls `./inbox/` (5s interval) → ingest text/media.\n2. Consolidation loop (30min) → if ≥2 unconsolidated, run ConsolidateAgent.\n3. HTTP API (`aiohttp` :8888): `/ingest`, `/query?q=`, `/consolidate`, `/status`, etc.\n4. Streamlit dashboard for UI.\n\n**Always-on:** asyncio tasks + Runner for continuous operation. Gemini 3.1 Flash-Lite for speed/cost.\n\n## 2.
